# وثيقة متطلبات المنتج (PRD) - تطبيق المدير الديناميكي للمحفظة الاستثمارية الشخصية

**الإصدار:** 1.1 (بعد دمج تحسينات Gemini AI)  
**تاريخ:** أبريل 2026  
**حالة:** معتمدة للتطوير (MVP)

---

## 1. ملخص تنفيذي

تطبيق جوال (iOS/Android) يقوم بدور مدير محفظة استثمارية شخصي ديناميكي شبه ذاتي، يتخذ قرارات الشراء والبيع المثالية بناءً على نموذج رياضي متطور (**Optimum Score - OS**) يجمع بين العائد المعدل بالمخاطر (Sharpe Ratio)، والانحراف السعري (Z-Score)، وتكلفة المعاملات. الهدف النهائي هو **تعظيم الربح (Profit Optimization)** مع احترام ملف المخاطر الشخصي للمستخدم.  
في نسخة MVP، يعمل التطبيق على **بيانات تاريخية فقط** (بدون APIs لحظية) ويقدم تنبيهات يدوية.

---

## 2. نطاق المنتج (MVP)

### يتضمن:
- تحديد الأهداف الاستثمارية الشخصية (4 أنماط رئيسية + مخصص)
- إدخال وإدارة الأصول (كمية، سعر شراء، تاريخ)
- تحليل التوزيع الحالي والمثالي لرأس المال باستخدام **نسبة شارب** ومصفوفة التغاير
- حساب Z-Score لكل أصل لتحديد مناطق التشبع البيعي والشرائي
- **نموذج Optimum Score (OS)** الموحد الذي ينتج إشارات شراء/بيع
- إشارات شراء (عند OS ≥ 0.7) وبيع (عند OS ≤ 0.3) مع اقتراح كمية وقيمة
- إعادة توازن استراتيجي مقترح بناءً على الانحراف عن الأوزان المستهدفة
- تحديث الأسعار يدوياً (لأنه لا توجد APIs لحظية في المرحلة الأولى)
- إشعارات دفع فورية عند ظهور الإشارات
- تصدير تقارير الأداء (PDF/CSV)

### لا يتضمن في الإصدار الأول:
- تنفيذ صفقات آلي
- تكامل مع حسابات وساطة
- بيانات لحظية (API حية)
- الاقتراض أو التمويل بالهامش
- Value Averaging المعقد

## 3. المتطلبات الوظيفية (Functional Requirements)

### 3.1 إعداد الملف الاستثماري (Investor Profile)

| المعرف | المتطلب |
|--------|----------|
| FR-01 | استبيان تفاعلي من 7 أسئلة لتحديد: تحمل المخاطر، الأفق الزمني، الهدف الأساسي |
| FR-02 | حساب درجة المخاطر (1-10) وعرض النمط المقترح: عنيف، متوازن، دخل ثابت، تأمين رأس المال |
| FR-03 | السماح بتعديل النمط يدوياً أو إنشاء نمط مخصص (نسب مئوية لكل فئة أصل) |

### 3.2 إدارة الأصول والمحفظة

| FR-04 | إضافة أصل جديد: الاسم، الفئة، الكمية، متوسط سعر الشراء، تاريخ الشراء |
| FR-05 | تحديث سعر الأصل الحالي يدوياً |
| FR-06 | عرض القيمة السوقية الحالية لكل أصل وللمحفظة الكلية |
| FR-07 | حساب نسبة توزيع المحفظة الحالية مقابل المستهدفة |

### 3.3 محرك التوزيع الأمثل لرأس المال (Capital Allocation Optimizer)

| FR-08 | حساب العائد المتوقع للمحفظة: \(E(R_p) = \sum w_i E(R_i)\) باستخدام متوسط العوائد التاريخية |
| FR-09 | حساب مخاطر المحفظة (التباين): \(\sigma_p^2 = \sum \sum w_i w_j Cov(R_i, R_j)\) |
| FR-10 | حساب نسبة شارب: \(S = \frac{E(R_p) - R_f}{\sigma_p}\) وتحسين الأوزان لتعظيمها |
| FR-11 | عرض الانحراف بين الوزن الحالي والوزن المستهدف |

### 3.4 محرك Optimum Score (OS) – المعادلة الذهبية

**المعادلة:**  
\[
OS_i = \alpha \left( \frac{E(R_i)-R_f}{\sigma_i} \right) + \beta (-Z_i) - \gamma C_i
\]  
حيث:  
- \(OS_i\): الدرجة المثالية للأصل \(i\) (0 إلى 1)  
- \(\alpha, \beta, \gamma\): أوزان ديناميكية (قيم ابتدائية: 0.4, 0.4, 0.2)  
- \(E(R_i)\): العائد المتوقع (من التاريخ)  
- \(R_f\): العائد الخالي من المخاطر (مثلاً 3%)  
- \(\sigma_i\): التقلب التاريخي  
- \(Z_i\): Z-Score = \((P_t - \mu)/\sigma\)  
- \(C_i\): تكلفة المعاملات (مثلاً 0.1% من قيمة الصفقة)

| FR-12 | حساب Z-Score لكل أصل باستخدام المتوسط المتحرك لـ 50 يوم والانحراف المعياري |
| FR-13 | حساب Optimum Score يومياً لكل أصل باستخدام المعادلة أعلاه |
| FR-14 | إذا \(OS \ge 0.7\) → إشارة شراء قوية |
| FR-15 | إذا \(OS \le 0.3\) → إشارة بيع قوية |
| FR-16 | عرض OS والمؤشرات المساعدة (Z-Score، العائد المتوقع، التقلب) لكل أصل |

### 3.5 تحديد حجم الصفقة (الكمية والقيمة)

| FR-17 | حساب قيمة الشراء المقترحة: \(\min(\text{rebalancing\_amount}, \text{cash} \times 0.3)\) |
| FR-18 | حساب كمية البيع المقترحة: إما لإعادة التوازن أو بنسب (25%، 50%، 100%) |
| FR-19 | عرض الاقتراح مع إمكانية تعديل الكمية يدوياً |

### 3.6 إعادة التوازن الاستراتيجي

| FR-20 | تنبيه لإعادة التوازن عند انحراف أي أصل عن وزنه المستهدف بأكثر من 5% (نسبة مطلقة) |
| FR-21 | عرض تفاصيل العملية: بيع كمية من أصل X، شراء كمية من أصل Y |

### 3.7 البيانات التاريخية (Backtesting – MVP)

| FR-22 | تحميل بيانات تاريخية من ملف CSV (يوفره المستخدم) للأصول المختارة |
| FR-23 | إعادة تشغيل محرك OS على البيانات التاريخية وعرض النتائج (عدد الإشارات، الربح الافتراضي) |
| FR-24 | تصدير تقرير باك تيست (PDF أو CSV) |

### 3.8 إضافات جديدة (FR-34 إلى FR-38) بعد دمج نموذج Gemini

| FR-34 | حساب Z-Score لكل أصل باستخدام المتوسط المتحرك لـ 50 يوم والانحراف المعياري (مكرر للتأكيد) |
| FR-35 | إذا Z-Score < -2 → تعزيز إشارة الشراء (يؤثر على OS) |
| FR-36 | إذا Z-Score > +2 → تعزيز إشارة البيع |
| FR-37 | حساب مصفوفة التغاير (Covariance Matrix) بين العوائد التاريخية للأصول لاستخدامها في تحسين المحفظة |
| FR-38 | استخدام نسبة شارب كدالة هدف لتحسين الأوزان في Capital Allocation Optimizer |

## 5. حالات الاستخدام الرئيسية (Use Cases)

### UC-1: مستخدم جديد يحدد ملفه الاستثماري
1. يجيب على استبيان المخاطر.
2. يعرض التطبيق النمط المقترح والأوزان المستهدفة.
3. يقبل المستخدم أو يعدل يدوياً.
4. يضيف الأصول (الكميات، أسعار الشراء).

### UC-2: استلام إشارة شراء مع كمية وقيمة مقترحة
1. يصل إشعار: "إشارة شراء لـ AAPL – OS=0.82".
2. يفتح التطبيق → يرى التفاصيل: السعر الحالي، Z-Score، الاقتراح (شراء 10 أسهم بقيمة 1850$).
3. يعدل الكمية (اختياري) ثم ينفذ الصفقة يدوياً خارج التطبيق.
4. يسجل الصفقة في التطبيق.

### UC-3: استلام إشارة بيع مع كمية مقترحة
1. إشعار: "إشارة بيع لـ BTC – OS=0.28".
2. يرى الاقتراح: بيع 0.05 BTC (نصف المركز) بقيمة 2600$.
3. يختار نسبة البيع (25%، 50%، 100%) أو يعدل يدوياً.
4. ينفذ البيع ويسجله.

### UC-4: تشغيل باك تيست
1. يرفع المستخدم ملف CSV بأسعار تاريخية.
2. يختار الفترة.
3. يعرض التطبيق الأداء الافتراضي: عدد الصفقات، العائد، مقارنة مع الشراء والاحتفاظ.

## 7. الملحق أ: المعادلات الرياضية الأساسية (LaTeX)

\[
\begin{aligned}
&\text{1. العائد المتوقع للمحفظة:} \quad E(R_p) = \sum_{i=1}^{n} w_i E(R_i) \\
&\text{2. تباين المحفظة:} \quad \sigma_p^2 = \sum_{i=1}^{n} \sum_{j=1}^{n} w_i w_j \text{Cov}(R_i, R_j) \\
&\text{3. نسبة شارب:} \quad S = \frac{E(R_p) - R_f}{\sigma_p} \\
&\text{4. Z-Score:} \quad Z = \frac{P_t - \mu}{\sigma} \\
&\text{5. Optimum Score (OS):} \quad OS_i = \alpha \left( \frac{E(R_i)-R_f}{\sigma_i} \right) + \beta (-Z_i) - \gamma C_i \\
&\text{6. RSI:} \quad RSI = 100 - \frac{100}{1+RS}, \quad RS = \frac{\text{avg gain}}{\text{avg loss}} \\
&\text{7. قيمة الشراء المقترحة:} \quad \text{BuyValue} = \min\left( (w_{target} - w_{current}) \cdot PV,\; \text{Cash} \times 0.3 \right) \\
&\text{8. كمية البيع المقترحة (إعادة التوازن):} \quad Q_{sell} = \frac{(w_{current} - w_{target}) \cdot PV}{P_{current}}
\end{aligned}
\]
